# AWS IAM User Management and MFA Security Lab

## Project Overview
This project demonstrates the implementation of AWS Identity and Access Management (IAM) security best practices by creating IAM users, assigning administrative permissions through groups, and enabling Multi-Factor Authentication (MFA) for enhanced account protection.

The lab was performed using the AWS Free Tier environment on Windows 11 and focuses on secure identity management within AWS cloud infrastructure.

---

# Objectives
- Understand AWS IAM fundamentals
- Create and manage IAM users
- Implement role-based access control using IAM groups
- Assign permissions using AWS managed policies
- Configure Multi-Factor Authentication (MFA)
- Verify secure IAM user login access
- Apply cloud security best practices

# Project Steps

## Step 1 — Creating an IAM User
A new IAM user named `Victory_Admin` was created inside AWS IAM.

### Actions Performed
- Navigated to IAM Dashboard
- Selected "Create User"
- Assigned console access
- Enabled autogenerated password
- Forced password reset at first login

### Purpose
This demonstrates how organizations create separate identities for users instead of sharing the AWS root account.

---

## Step 2 — Creating an IAM User Group
An IAM user group named `SOC_Analysts` was created.

### Actions Performed
- Created a new IAM group
- Attached the `AdministratorAccess` policy
- Added the `Victory_Admin` user to the group

### Purpose
This demonstrates Role-Based Access Control (RBAC) and centralized permission management.

---

## Step 3 — Reviewing User Permissions
The IAM configuration was reviewed before final creation.

### Permissions Verified
- IAMUserChangePassword
- SOC_Analysts Group Membership
- Administrative Privileges via Group Policy

### Purpose
This step validates correct permission assignment and user configuration.

---

## Step 4 — Retrieving IAM User Credentials
AWS generated login credentials for the new IAM user.

### Information Generated
- Console Sign-in URL
- IAM Username
- Temporary Password

### Security Note
Sensitive information such as account IDs and credentials should always be blurred before public documentation.

---

## Step 5 — IAM User Login Verification
The newly created IAM user successfully logged into the AWS Management Console.

### Verification Performed
- Entered AWS Account ID
- Used IAM Username
- Logged in with generated password
- Accessed AWS Console Dashboard

### Purpose
This confirms that the IAM user account was configured correctly and can securely access AWS resources.

---

## Step 6 — Configuring Multi-Factor Authentication (MFA)
MFA was configured using an authenticator application.

### Actions Performed
- Created MFA device named `Victory_MFA`
- Selected "Authenticator App"
- Scanned AWS-generated QR code
- Entered two consecutive MFA codes
- Successfully linked virtual MFA device

### Security Benefits
- Protects against unauthorized access
- Reduces phishing risks
- Adds a second authentication factor
- Strengthens cloud account security

---

## Step 7 — MFA Verification
AWS confirmed successful MFA activation for the IAM user.

### Verification Results
- MFA device assigned successfully
- Secure login enabled
- IAM account protected with two-factor authentication

### Purpose
This demonstrates cloud identity hardening and AWS security best practices.
